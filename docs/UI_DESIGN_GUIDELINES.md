# UIデザインガイドライン

## 🎯 目的
このドキュメントは、アプリケーションのユーザーインターフェース（UI）に一貫性を持たせ、可用性・アクセシビリティ・ブランドイメージを高めるための指針をまとめたものです。

---

## 1. ブランドとトーン
### 1.1 ブランドの方向性
- 清潔感と信頼感を重視したデザイン
- シンプルで直感的な操作性

### 1.2 トーン＆マナー
- フレンドリーかつプロフェッショナル
- 専門用語は必要最小限にし、説明文はわかりやすく

---

## 2. カラー
### 2.1 ベースカラー
- **Primary（濃い青）**: `#0052CC`（基調：青）
- **Secondary（淡い青）**: `#00B4FF`（ふんわりした水色）

### 2.2 テキストカラー
- **本文テキスト**: `#252525`（しっかり読める黒）
- **セカンダリテキスト**: `#4A4A4A`

### 2.3 アクセントカラー
- **アクセント（赤）**: `#E60012`（超会議ロゴを意識した赤）
- **追加アクセント（ニコニコっぽさ）**: `#FF7F00`（ニコニコ動画の暖かいオレンジ寄り）

### 2.4 状態カラー
- **成功**: `#28A745`
- **警告**: `#FFC107`
- **エラー**: `#DC3545`
- **情報**: `#17A2B8`

---

## 3. タイポグラフィ
### 3.1 フォント（Google Fonts）
- **メインフォント（見出しなど）**: `LINE SEED`（Google Fontsから読み込み）
- **サブフォント（本文など）**: `MPLUS Rounded`（Google Fontsから読み込み、柔らかさを補強）

#### 3.1.1 読み込み例（CSS）
```css
@import url('https://fonts.googleapis.com/css2?family=Line+Seed+JP:wght@400;600;700&display=swap');
@import url('https://fonts.googleapis.com/css2?family=M+PLUS+Rounded+1c:wght@400;500;700&display=swap');

:root {
  --font-heading: 'Line Seed JP', system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', 'Helvetica Neue', Arial, 'Noto Sans', sans-serif;
  --font-body: 'M PLUS Rounded 1c', system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Roboto', 'Helvetica Neue', Arial, 'Noto Sans', sans-serif;
}
```

### 3.2 フォント階層（例）
- **見出し H1**: 32px / 700（`--font-heading`）
- **見出し H2**: 24px / 600（`--font-heading`）
- **本文**: 16px / 400（`--font-body`）
- **小さい本文**: 14px / 400（`--font-body`）

---

## 4. レイアウトとスペーシング
### 4.1 グリッド
- ベースグリッド: 8px 単位

### 4.2 マージン・パディング
- 小: 8px
- 中: 16px
- 大: 24px

### 4.3 レスポンシブ（PC / スマホ） — **必須**
- ブレークポイント（例）
  - モバイル: up to 640px
  - タブレット: 641px〜1024px
  - PC: 1025px〜
- レイアウトは **モバイルファースト**で設計（初回からモバイル利用を前提とする）
- 横幅が狭いときはサイドバーや補助パネルを折りたたむ／ドロワー化
- タッチ操作を考慮し、ボタンやリンクのタップ領域は最小44x44pxを確保
- フォントサイズ・行間はデバイス幅に応じて調整（流動的なサイズ、`clamp()` 推奨）

---

## 5. コンポーネントガイド
### 5.1 ボタン
- **プライマリボタン**: 背景カラー=プライマリカラー、文字色=白
- **セカンダリボタン**: 背景=透過・枠線あり、文字色=プライマリカラー

### 5.2 フォーム入力
- 角丸: 4px
- フォーカス時: 境界線をプライマリカラーに
- エラーメッセージ: テキストカラー=エラーカラー

### 5.3 モーダル
- 背景オーバーレイ: `rgba(0,0,0,0.5)`
- コンテナ: 最大幅 600px、パディング 24px

---

## 6. アクセシビリティ
- コントラスト比: WCAG AA（最低 4.5:1 以上）
- フォーカスインジケーター: 明確に見えるアウトラインを付与
- キーボード操作: 全てのインタラクティブ要素が Tab で操作可能

---

## 7. アイコン・イラスト
- Font Awesomeを使用可能とする
- アイコンはシンプルでわかりやすい表現
- すべて同じスタイル（線幅・角丸）に統一

## 10. インタラクションとフィードバック
### 10.1 反応の速さ
- クリック/タップに対して 100ms 以内にフィードバックを返す
- 遅延が発生する場合は、ローディングスピナーやプログレスバーを表示する

### 10.2 トランジションとアニメーション
- アニメーションは短く（200〜300ms）、目的を補強するものに限定する
- アニメーションはユーザーの注意を奪わないようにする
- アニメーションをオフにする設定を検討する（OSやブラウザの「省電力モード」や「減弱モード」を尊重）

### 10.3 フィードバックの一貫性
- 成功/エラー/警告は色とアイコンで一貫させる
- キーボード操作でも同等のフィードバックが得られるようにする

---

## 11. エラーと状態管理
### 11.1 エラーハンドリング
- 具体的で行動可能なエラーメッセージを表示する
- API 呼び出しやネットワークエラー時はリトライ手段を提供する
- フォームは送信失敗時に入力内容を保持し、ユーザーがやり直せるようにする

### 11.2 空状態（Empty State）
- データがない場合は、ガイドメッセージと次に取るべきアクションを明示する
- 例: 「まだセッションがありません。新しいセッションを作成してください。」

### 11.3 ローディング状態
- データ読み込み中はプレースホルダーやスケルトンスクリーンを使用する
- 長時間かかる処理には進行状況を示す表示を追加する

---

## 12. ユーザー調査と改善
- 定期的にユーザビリティテストを行い、実際のユーザーの操作を観察する
- フィードバックは優先度をつけて改善に反映する
- 変更による影響を測定するために、主要な指標（例: 完了率、途中離脱率）を追跡する

---

## 13. ドキュメント化
- UI コンポーネントやパターンの実装例（コードスニペット）をドキュメントに追加する
- デザインと実装の間に乖離が発生した場合は、ドキュメントを最優先で更新する
